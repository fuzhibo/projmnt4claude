# init-requirement - Natural Language Task Creation

Automatically analyze natural language descriptions and create structured tasks. No manual form-filling needed вҖ” one sentence converts a requirement into a task.

## Important: Task Persistence Requirement

**This command must complete the entire flow to ensure the task is persisted to `.projmnt4claude/tasks/`.**

After running this command:
1. CLI displays requirement analysis results (auto-extracts priority, role, complexity, checkpoints)
2. CLI asks for user confirmation (auto-confirmed in non-interactive mode)
3. CLI creates task files (meta.json and checkpoint.md)
4. CLI asks whether to add to execution plan (skipped in non-interactive mode)

## Prerequisites

Initialize the project before using this command:
```bash
projmnt4claude setup
```

## Usage

### Interactive Mode (default, for human users)
```bash
node ${CLAUDE_PLUGIN_ROOT}/skills/projmnt4claude/dist/projmnt4claude.js init-requirement "<requirement description>"
```
Interactive mode guides you through confirming analysis results, editing title/description/priority/role, and choosing whether to add to the execution plan.

### Non-Interactive Mode (recommended for AI agents)
```bash
# Use -y or --yes to skip all confirmations
node ${CLAUDE_PLUGIN_ROOT}/skills/projmnt4claude/dist/projmnt4claude.js init-requirement -y "<requirement description>"

# Also skip the "add to plan?" prompt
node ${CLAUDE_PLUGIN_ROOT}/skills/projmnt4claude/dist/projmnt4claude.js init-requirement -y --no-plan "<requirement description>"

# Skip checkpoint quality validation
node ${CLAUDE_PLUGIN_ROOT}/skills/projmnt4claude/dist/projmnt4claude.js init-requirement -y --skip-validation "<requirement description>"
```

## Options

| Option | Description |
|--------|-------------|
| `-y, --yes` | Non-interactive mode: skip all confirmations, create task directly with analysis results |
| `--no-plan` | Don't ask whether to add the task to the execution plan |
| `--skip-validation` | Skip checkpoint quality validation (not recommended) |
| `--template <type>` | Description template type: simple (default) or detailed (structured) |

## Auto-Analysis Features

| Analysis Dimension | Description | Example Keywords |
|-------------------|-------------|-----------------|
| Priority | P0-P3 four levels | urgentвҶ’P0, importantвҶ’P1, optionalвҶ’P3 |

## FAQ

**Q: What if I see "project not initialized"?**
A: Run `projmnt4claude setup` first to initialize the project management directory.

**Q: What if the analysis results are inaccurate?**
A: In interactive mode, you can manually edit each field. In non-interactive mode, use `task update` to modify afterwards.

**Q: How to view created tasks?**
A: Run `projmnt4claude task list` to see all tasks.

**Q: How to modify a created task's priority?**
A: Run `projmnt4claude task update <taskId> --priority P1`.
